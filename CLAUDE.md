## Working with markdown docs

1. **Re-`Read` immediately before any `Edit`.** Even if you read the file earlier in the conversation, read it again right before editing so your `old_string` matches what's actually on disk.
2. **If the file looks different than you expected, stop and ask.** Unfamiliar sections, reordered content, or new headings likely mean the user edited it — don't try to "reconcile" by rewriting.
3. **Prefer appending to restructuring.** For running notes (logs, journals, findings), add new dated entries at the bottom rather than reorganizing existing content.

## Writing

- Write concisely. Avoid using unneccesary prose. Communicate the most important ideas in as few words as possible.
- Leave out information that is implied, obvious, or unnessecary.
